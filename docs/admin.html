Admin / Approval

Submissions are created as GitHub Issues labeled submission. When you review an issue you can either add the approve label or click the Approve link in the notification email (it pre-fills a comment with /approve).

  • When an issue is opened, a workflow will email the owner with the submission and links to the issue (including quick approve/reject comment links).
  • To approve: add the approve label or post a comment `/approve`. The approval workflow will append the project to /data/projects.csv, regenerate /data/projects.json, and open a pull request. When the PR is merged, the originating issue is closed automatically.
  • To reject: add the reject label or post a comment `/reject`. The workflow will comment and close the issue.

Security notes

Only repository collaborators can add labels. Workflows check the actor against the configured OWNER_USERNAME secret to ensure only the owner can approve.

See README for full setup steps (secrets, email SMTP, enabling Pages).
